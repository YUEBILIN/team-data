# 阶段性进展同步与待决策议题

> 日期：2026-04-20
> 作者：tangkun-tank998

## 一、已完成

核心测试链路已打通，具备端到端可用能力：

- **通道**：飞书通道测试已跑通
- **用例管理**：支持 case 文件夹上传、编辑、删除
- **测试执行**：支持单次执行与多次执行
- **指标统计**：响应时间、Token 消耗统计，并支持多次运行之间的对比

## 二、待完善

以下功能已进入待办，尚未开工或未完成：

- 报告中心
- 国际化支持
- 页面 UI 优化
- 平台管理员功能优化
- 分享页广告位 / 内容

## 三、需要讨论决定

### 重点议题

1. **产品定位**（重点）
   bench-saas 究竟只做一个 benchmark 测试平台，还是可以向其他产品类型扩展？
   目前 bench-saas 已具备 LLM 统一接口能力，是否顺势扩展为类似 OpenRouter 的网关产品，以完善 Enclaws 家族生态？

2. **收费模式**（重点）
   用户在平台上运行测试时，Agent 实际访问的是我们平台托管的大模型，这部分成本如何对用户计费？按 Token？按 Case 次数？按订阅？

### 次级议题

3. **直连通道**：不依赖任何三方 IM，直接通过 API 向 OpenClaw / EnClaws 发消息进行测试
4. **微信、钉钉测试通道**
5. **在线编写 case**：在 Web 端直接编辑测试用例
6. **LLM 编写测试用例**：自然语言描述 → 结构化 case
7. **LLM Judge**：用 LLM 对测试结果进行自动化判定

## 四、希望得到的反馈

- 重点议题 1、2 希望在本周内有明确方向，后续功能优先级依赖这两项决策
- 次级议题可排入 Phase 1 / Phase 2 的节奏讨论
