### 第一层：产品宏观定义（对外沟通版）

**定位**：面向"一人公司（OPC）"的协作平台，通过 AI 工具+平台生态，让 1 人完成原本 5 人团队的工作，安全高效地撮合链主企业与 OPC 的交易。

**三类核心用户 & 关键动线**
| 用户 | 核心痛点 | 关键能力 | 主操作动线 |
|---|---|---|---|
| **链主企业（发包方）** | 碎活外包易失控、验收标准模糊 | AI 生成标准化 SOW → 智能匹配 OPC → 全程履约监控 → 多维验收 | 发起需求 → AI 产 SOW → 选定 OPC → 监控/验收 → 放款复盘 |
| **OPC 创业者（接单方）** | 获单不稳定、产能受限、合规门槛高 | 项目大厅 → AI 接包风控 → AI 秘书调度 → 政策/财税/法务支持 | 找单 → 风控评估 → 报价/中选 → AI 辅助履约 → 交付收款 |
| **平台运营（数科院/园区）** | 生态运营黑盒、缺乏决策抓手 | 运营驾驶舱 → OPC 准入/评级/淘汰 → 公共活动与转化追踪 | 看数据大盘 → 审批/干预 → 组织活动 → 复盘生态健康度 |

**核心设计原则（对外沟通精简版）**

- 流程按业务对象组织，不按 AI 能力切分
- AI 是"地图"不是"裁判"：所有 AI 输出附客观依据，最终决策权归人
- 一事项一时间线：全生命周期记录不被切碎到独立页面

### 第二层：五一前 MVP 基线（交付边界 & 验收标准）

**目标**：1.5 天内跑通"发单→接单→履约→验收"主链路骨架，确保数科院可见、可操作、可验收。

| 页面             | 状态      | 说明与理由                            |
| -------------- | ------- | -------------------------------- |
| 需求创建页          | 🔵 真功能  | 基础表单（标题/简述/预算/周期/标签），支撑发包起点      |
| 项目大厅           | 🔵 真功能  | 列表展示已发布需求，接单方入口                  |
| 需求详情页（时间线）     | 🔵 真功能  | 纵向 Activity Feed 承载状态流转，核心载体     |
| 报价提交/操作区       | 🔵 真功能  | 交易核心动作（提交/接受/拒绝/完成）              |
| 角色切换（登录替代）     | 🔵 真功能  | 前端下拉模拟发包/接单， bypass 复杂鉴权         |
| AI 需求澄清        | ⚠️ 演示态  | 占位入口，1.5天接 LLM 调优风险极高，首期仅展示 UI   |
| 智能匹配 OPC       | ⚠️ 演示态  | 静态 mock 卡片（头像/标签/mock 评分），无真实画像库 |
| AI 监控/验收/法务/财税 | 🚫 明确不做 | 强依赖 Agent/规则引擎/第三方数据，短期不可交付      |

**已知缺陷（由边界划分直接导致）**

1. 无真实 AI：SOW 全靠手填，无多轮澄清/智能生成
2. 无权限隔离：发包/接单靠前态切换，无后端租户隔离
3. 匹配逻辑缺失：OPC 推荐为静态数据
4. 无业务规则校验：预算/周期合理性不检查
5. 无文件/合同流：不支持附件与凭证留存
6. 验收纯手动：点"完成"即结束，无多维指标或证据链

**验收标准（数科院对齐版）**

- 主链路跑通：创建→发布→大厅可见→报价→接受→状态变履约中→时间线刷新→点完成→状态变已完成
- 无阻断缺陷：全流程无 500/白屏/表单丢数据
- 时间线一致性：操作后 3 秒内新增对应卡片
- 加载性能：核心页首屏 <2s（测试环境）
- 演示可用性：数科院代表 5 分钟内不依赖指导独立走完

### 第三层：两周迭代优先级

| 优先级    | 能力                 | 理由                                         |
| ------ | ------------------ | ------------------------------------------ |
| **P0** | AI 需求澄清（单轮/少轮 LLM） | 替代手填 SOW，产品核心价值点；调 API 不涉 Agent，2-3 天可出可用版 |
| **P1** | 完整事项时间线            | 补齐报价/接受/里程碑/验收等卡片，决定体验质感与差异化               |
| **P2** | 候选 OPC 基础画像        | 将 mock 换为真实查询（标签/单量/评分），开发量小但显著提升决策感       |
| **P3** | AI 输出人工确认/编辑       | SOW/匹配结果可二次修改，解决 009 指出的 AI 主观性风险          |
| **P4** | 简单履约里程碑 & 站内通知     | 支持进度更新与关键动作提醒，完善协作闭环，非阻塞但提升真实感             |

### 第四层：长期产品规划

承载 012 中的生态愿景，**不作为短期承诺，仅作战略方向同步**：

- **运营驾驶舱**：全链路漏斗、健康度指标、待决策项聚合
- **AI 员工市场**：法务/财税/政策/监控/验收等角色动态调度
- **动态评级体系**：基于履约/验收/申诉的持续评级，替代一次性认证
- **多业态适配**：主流程通用 + AI 按业态生成 SOW/监控点/验收标准
- **仲裁与保证金**：SOW 异议处理流程 + 极简兜底机制
- **多公司切换 & 上下文延续**：RAG/多 Agent 隔离成熟后实现
- **社区活动 & 政策申报**：沙龙/培训/算力券/补贴转化追踪
