# OPC-数字员工服务台-产品规划（001）

本 thread 是opc数字员工服务台产品线主话题。本帖汇总 ai.xhopc.com 竞品 + 数科院 6 份参考材料的市场调研。002 发布 PRD。

## 一、竞品 · ai.xhopc.com

**四方结构与我方一致**：园区 / 企业 / OPC / 平台方；竞品没有给咨询方产品位置——印证行业共识。

**可借鉴 8 条**：公司管家（→ 秘书 agent）/ 注册三态（模拟/真实/待注册）/ 资源中心商机状态机 / 技能库 GitHub 链接导入 / 待办三线视图（生存优先/关键推进/后续安排）/ 商业画布结构化输入 / 用量明细按任务会话统计 / 记忆与成长 UI 参考。

**不借鉴 6 条**：Token→算力中间抽象（掩盖真实成本）/ 模型限国产默认（违用户选择权）/ 物联网数字孪生（花架子）/ 资金闭环（政府话术不进产品）/ 炫技命名（如"龙宫蜂群"）/ 咨询方作为产品角色（违四方边界）。

完整借鉴落地点和不借鉴理由有本地文件，需要的同学找我要。

## 二、合作伙伴资料 · 数科院 6 份

**有效信息**（按价值密度排）：

- **政策数据**：省级 2027 目标 30+ 社区 / 1000+ OPC / 5000P 算力；市级 20 社区 / 100 产品 + 算力券 60%/500 万上限 / 模型调用补贴 30%/100 万上限 / 社区运营 100 万/年
- **产业锚点**：青羊区文旅 204 亿文创产值 + 36.65 亿春节档；核心链主成都传媒集团、天府城发、宽窄巷子 IP 等
- **订单分级模型**：A 5-20 万 / B 1-5 万 / C ≤5 千；链主释单 3-5% 专项补贴
- **OPC 分层概念**：T1 / T2 / T3（可作入驻资质参考，本地化标准另定）
- **全国 OPC 体量**：1200 万+ 从业者、2025 新增 286 万（同比 +47%）、115 个建成社区

**过滤红线**（不进产品）："社区伙伴责任共担" / "揭榜挂帅" 等政府话术、"三方/五方/七方" 结构设计、"走访 5000 企业 / 100 家 OPC 招募" 等咨询方自身运营动作——这些是数科院作为咨询方的话术和执行动作，不影响我方产品结构。

## 三、对产品方向的影响

两份材料合起来印证/影响产品方向：

- **四方结构确认**：质数（技术方）/ 园区（运营方）/ 企业（订单方）/ OPC（服务方）。数科院等合作方以外部渠道角色存在，不进产品结构
- **商机来源主干**：竞品验证的公开招投标抓取 + 运营方推送订单 + 企业自发发单
- **套餐叠加**：ToG 场景用政府算力/模型补贴摊薄用户 Token 成本（成都市 30%/100 万 + 60%/500 万可叠加）
- **Phase 1 聚焦内容工作室**；青羊文旅 / 金牛电商直播/数据标注/生物医药 等为 Phase 2 场景复用候选
