# 数科院合作 AI-OPC 项目 ROI 测算

## 0. 测算说明

**目的**：
1. 为老板决策商务底线提供数据支撑（能接的最低分账率、政府资源怎么要、独家期限）
2. 校准蒋鲲 4/26 向青羊区政府汇报时的 ROI 口径，避免数字失真

**口径**（老板拍板）：
- **全量口径**：Phase 1 内容工作室 + Phase 2 数字化协同平台研发投入**全部算入**成本
- **收益端只算数科院合作带来的**：订单分账 + 政府资源（基金 / 补贴券 / 模型调用补贴）。**不计入**对外自营 SaaS 订阅收入
- **研发提供两套估算**（AI 辅助开发 vs 传统开发，让老板看到两条路径差距）

**置信度标注**：
- 成都 OPC 市场数据：**高**（近期新闻 + 官方政策，见 6.4）
- 参数假设：中（行业通用 + ken 规划 + 蒋鲲 PPT 互相印证）
- 青羊区实际数据：**低**（搜索未发现，用其他成都区数据代理）

### 1.5 Token 成本的重新设计（免费额度 + 超额自费）

**用户直觉的定价洞察**（关键）：OPC 老板群体 token 消费分布**极度不均**——赚得少的一分都不想出，赚得多的 1000-2000 都愿意出。平均值无意义。

**对应的产品设计**：

- **基础免费额度**（我们承担）：日上限 50k standard + 10k senior（覆盖 zhangbo 排班大约 70% 的需求）
- **超额**：用户自选两条路径
  - 路径 A：**订阅付费升级**（¥99/¥199/¥399 三档）—— 用户付订阅费给我们，但 token 消耗仍由我们付给模型方；**我方仍承担 token 成本**，订阅收入用于覆盖并超过成本（毛利正）
  - 路径 B：**自配 API Key**（豆包 / DeepSeek / 千问）—— 用户直连模型方、自担 token 成本，**我方彻底不承担**
- 数科院合作场景下的特殊考虑：青羊区入驻 OPC 可能期待"园区配套免费"，走不通路径 A 的订阅升级；主要靠路径 B（引导自配 API Key） + 政府模型调用补贴（成都市 30%/100 万政策）摊薄我方成本

**悲乐区间（我方净承担）**：

- **悲观 ¥120/OPC/月**：免费额度设计宽松 + 数科院场景 OPC 期望全免费、路径 B 转化低 → 我们承担比例高
- **乐观 ¥20/OPC/月**：路径 B 转化高 + 拿到成都市模型调用补贴 → 我们承担比例低

⚠ 仍属假设，Daisy 种子用户跑一个月就可以回标真实数字。

## 5. 敏感性分析

按**对年净利润的弹性**排序（中位档基础上单独变动 ±50%）：

| 参数 | ±50% 对年净利润影响 | 敏感性 |
|---|---|---|
| **是否拿到政府基金** | ± ¥750,000 | **极高** |
| **入驻 OPC 规模** | ± ¥76,800 ~ ± ¥1,587,000（看是否有政府资源）| **极高** |
| **单笔订单均值** | ± ¥76,800 | 高 |
| **分账率** | +50%（3→5% 区间）| 高 |
| **成交率** | ± ¥76,800 | 高 |
| **研发模式**（传统 vs AI 辅助）| ± ¥64,000 | 中 |
| **运营人力** | ± ¥75,000 | 中 |
| **Token 成本**（免费额度模式）| ± ¥3,000 ~ ± ¥12,000 | **低** |
| **4/26 准备人天** | ± ¥2,750 | 极低 |

**结论 1**：**政府资源（基金 + 补贴券 + 模型调用补贴）是 ROI 成立的决定性因素**——没它哪怕乐观参数都不容易回本；拿到中位档就盈利。

**结论 2**：**规模侧（入驻数 × 活跃率）是我们几乎无法直接控制的变量**，依赖数科院对接能力 + 青羊区园区实际招商进展。

**结论 3**（v2 改写）：**技术侧优化的分档取舍原则**：
- **如果能把 token 消耗做到数量级降低**（比如月均从 ¥120 → ¥12，降 10x）→ **必须做**，这等价于对规模侧乘数放大，规模翻倍时成本不会线性涨
- **如果只是 20-30% 的小幅优化** → 不值得做，敏感性分析显示对年净利润影响 < ¥5 万/年，精力放在商务侧更划算
- 免费额度 + 用户自配 API 的设计本身就是把"数量级降低"变成了产品机制，不用靠技术硬 KO

## 6. 关键 takeaway 与商务建议

### 6.1 从 ROI 看"要不要接合作"

**结论**：接，但必须**把政府资源锁定为合作生效条件**。

- 无政府资源：悲观档血亏、中位档也亏；只有乐观档勉强盈利（纯分账模式）
- 有政府资源：中位档盈利（¥87 万）、乐观档千万级回报

### 6.2 商务底线建议（深度版）

> **判断框架**：商务底线 = f(刚性成本 × 机会成本 × 我方最佳备选 × 对方在意什么)。每条条款先问 4 个问题：
>
> 1. 低于什么数我们刚性亏损？
> 2. 低于什么数这精力不如做别的？
> 3. 如果谈崩，我们另有什么路？（即 **最佳备选** — 谈判破裂后的替代方案）
> 4. 对方最想保的是什么？这是我们的杠杆点。

---

**① 分账率**

| 档位 | 值 | 依据 |
|---|---|---|
| 绝对底线 | **3%** | 蒋鲲 PPT 公开区间下限。低于 3% = 他单方压价，我们有权拒 |
| 理想目标 | **5%** | PPT 上限。PMF 后的可持续价位 |

**为什么这是底线**：中位档 3% + 无政府资源 = 年亏 33 万；3% + 有政府资源 = 年赚 81 万——3% 是"绑定政府资源后的"底线，不是独立底线。

**谈判杠杆点**：
- 引用"国内第三方支付通道 4-6%"，我们 3-5% 低于行业通道费率已经让利
- 我们做了全套工具栈 + 数字员工，不是单纯支付通道

**最佳备选**：如果谈不到 3%，我们走自营 SaaS 路径（对外卖订阅），不接合作 —— 所以分账率确实有下限，不能让步到 2% 以下

---

**② 政府专项基金**

| 档位 | 值 | 依据 |
|---|---|---|
| 绝对底线 | **条件+金额必须明确**（不接受"尽力争取"）| 模糊承诺 = 空头支票，无法 justify 投入 |
| 理想目标 | **¥500 万分 2 年** | PPT 500-1000 万下限 |
| 超额目标 | **¥1,000 万分 2-3 年** | PPT 上限，Phase 2 进入后可谈 |

**为什么这是底线**：敏感性分析显示政府资源 ±50% 影响净利润 ±¥75 万/年，这一条决定项目赚不赚钱

**谈判杠杆点**：
- 对政府讲"杠杆故事"：社会资本 500 万（我们研发投入）+ 政府 500 万 = 1000 万撬动千亿产业，乐观档 10-50 倍杠杆
- 让数科院做合作生效条件担保——基金不到位合同不生效

**最佳备选**：无政府资源不做——这不是可谈的，是硬条件

---

**③ 入驻 OPC 保底规模**

| 档位 | 值 | 依据 |
|---|---|---|
| 绝对底线 | **年内 30 家** | 成都高新区首发 12 / 绵阳首个 25 / 对青羊区合理要求 30 |
| 理想目标 | **年内 100 家** | 对标高新区 2026 年目标 |
| 超额目标 | **3 年 200+ 家** | 对标高新区 3 年 1000+，取 20%+ |

**为什么这是底线**：低于 30 家，哪怕 5% 分账 + 500 万基金都难以 5 年内回本

**谈判杠杆点**：
- 青羊区政府对 OPC 创业社区本身有 KPI（成都市 2027 年底 20 家基地）
- 我们做第一个深度合作平台，就能绑定区级 KPI 成为示范点

**最佳备选**：如果青羊区招商能力不到 30 家，我们转向其他区（高新区已有成熟生态）

---

**④ 独家排他**

| 档位 | 值 | 依据 |
|---|---|---|
| 绝对底线 | **Phase 2 协同平台启动后 1 年独家** | 协同平台投入 75-115 人天，1 年独家是合理窗口 |
| 理想目标 | **Phase 2 + Phase 3（3 年独家）** | 软件行业标准独家条款 |

**为什么这是底线**：我们承诺 Phase 1+2 研发 140-220 人天（¥15-24 万投入），如果青羊区同时对接别的平台，用户被稀释，我们投入打水漂

**谈判杠杆点**：
- "我们投入 X 万研发换 1 年独家窗口，对政府是合理保护创新资本的方式"
- 作为回报，承诺保底入驻转化率（比如 70% 的入驻 OPC 年内上平台）

**最佳备选**：不独家 = 同样的研发投入按比例缩减（只做内容工作室不做协同平台）；这样就不用接受无独家保护

---

**⑤ 模型调用 / 算力补贴**

| 档位 | 值 | 依据 |
|---|---|---|
| 绝对底线 | **明确成为指定服务商 + 有申报路径** | 政策里有补贴但需要通过路径 |
| 理想目标 | **¥100 万 2 年**（成都市政策）| 蒋鲲 PPT 提及 |
| 超额目标 | **类天府新区级 ¥500 万/年算力补贴** | 天府新区自研大模型政策 |

**为什么这是底线**：如果拿不到补贴，Token 成本 + 算力全由我们/用户承担，悲乐差距巨大

**谈判杠杆点**：
- 数科院作为方案提交方协助我们拿认定
- 把算力补贴流向写进方案（流向我们最终受益的是全部入驻 OPC）

**最佳备选**：无补贴时通过免费额度 + 用户自配 API 的产品设计转嫁成本（Token 设计已天然覆盖这部分风险）
