# bug修改-AI助手遇到登录态失效时应提示重新登录而不是显示invalid_token

AI 助手遇到登录态失效时，应提示重新登录，而不是显示 invalid_token

**标题**

AI 助手遇到登录态失效时，应提示重新登录，而不是显示 `invalid_token`

**正文**

目前生产环境里，用户在 Web UI 调用 AI 助手时，如果登录态已经失效，前端会直接显示：

`错误：invalid_token`

但从生产日志看，这类错误并不是 AI provider、模型配置或 API Key 的问题，而是 `/api/ai/.../chat` 在进入 AI 调用逻辑前，被后端鉴权拦截并返回了 `401 {"detail":"invalid_token"}`。随后同一用户访问 `/me` 也会返回 `401`，重新登录后 AI 调用恢复正常。

这个体验容易误导用户，也会让排障方向偏向 AI 配置。建议调整为：

1. 当前端调用 `/api/ai/.../chat` 收到 `401 invalid_token` 时，不展示原始错误。
2. UI 提示应改为类似：`登录已失效，请重新登录后继续使用 AI 助手。`
3. 可以直接引导用户跳转登录页，或提供“重新登录”按钮。
4. 其他业务接口如果也遇到同类 `401 invalid_token`，也应尽量统一处理为登录态失效，而不是暴露底层错误码。
5. 真正来自 AI provider 的错误，仍然保留在 AI 错误路径里展示，例如 provider 返回 401、模型不存在、余额不足、网络超时等。

验收标准：

- 登录态失效后点击 AI 发送，不再显示 `invalid_token`。
- 用户能明确知道需要重新登录。
- 重新登录后回到原页面，AI 助手可以继续使用。
- 后端日志中仍然保留真实状态码，方便工程排查。
