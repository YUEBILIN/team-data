# UI优化需求-Mardown格式的渲染

## 背景

目前 Pivot 使用内置的 Markdown 渲染风格来展示文档。但不同用户对 Markdown 阅读样式的偏好不同，例如字体大小、行距、标题样式、代码块样式、引用样式等。

为了提升阅读体验，Pivot 应当允许用户在多种 Markdown 渲染风格中进行选择，而不是所有人都固定使用同一种系统默认样式。

## 目标

支持 Markdown 渲染风格配置，让系统管理员可以设置默认渲染风格，同时允许每个用户选择自己的个人默认渲染风格。

## 需求描述

Pivot 需要内置 3 到 5 个常用 Markdown 渲染风格。

系统管理员可以在管理后台配置整个 Pivot 系统的默认 Markdown 渲染风格。这个默认风格适用于尚未设置个人偏好的用户。

在每个 thread 的顶部，用户可以看到 Markdown 渲染风格切换入口，并从系统内置的多种风格中选择一种。

用户一旦在任意 thread 中切换渲染风格，该选择将覆盖系统管理员设置的默认风格，成为该用户自己的默认渲染风格。

用户的默认渲染风格对该用户在整个 Pivot 中查看的所有 thread 生效，而不是只对当前 thread 生效。

## 渲染优先级

Markdown 渲染风格的生效优先级为：

1. 用户个人选择的默认渲染风格。
2. 系统管理员配置的系统默认渲染风格。
3. 如果管理员未配置，则使用 Pivot 内置的初始默认风格。

## 期望结果

不同用户可以按照自己的阅读偏好选择 Markdown 渲染风格。

管理员可以控制系统的默认阅读体验，但不会限制用户的个人选择。

用户在任意 thread 顶部切换一次渲染风格后，后续查看 Pivot 中所有 thread 都会使用该风格，形成一致的个人阅读体验。

## 交给实现同学思考的问题

- 内置的 3 到 5 个 Markdown 渲染风格分别应该覆盖哪些典型阅读偏好。
- 渲染风格配置应该保存到用户设置表，还是复用现有用户偏好配置。
- thread 顶部的切换入口应该是下拉选择、分段按钮，还是更轻量的菜单。
- 切换渲染风格后是否需要立即刷新当前文档预览。
- 管理员修改系统默认风格时，是否只影响未设置个人偏好的用户。
