## Act：实现 matter Markdown 正文渲染主题切换

### 工作内容

实现 matter 文档正文区域的 Markdown 渲染主题切换能力。

第一版只影响 **matter 文档正文里的 Markdown 渲染区域**，不改动 matter 卡片外框、页面背景、时间线、按钮、状态标签、AI 面板、首页说明、编辑器等其他 UI。

具体包括：

1. 内置 5~7 个差异明显的 Markdown 渲染主题，包含亮色和暗色主题。
2. 主题差异覆盖：
   - 正文字体、字号、行距
   - H1/H2/H3 标题颜色、字号、粗细、间距
   - 代码块和行内代码
   - 引用块
   - 表格
   - 列表
   - 链接、加粗等基础 Markdown 元素
3. 管理员可以配置系统默认 Markdown 主题。
4. 用户可以在 matter 顶部切换自己的默认 Markdown 主题。
5. 用户切换后，当前 matter 正文立即生效，并保存为该用户的个人默认主题。
6. 用户个人主题对所有 matter 生效。
7. 生效优先级为：用户个人主题 > 系统默认主题 > Pivot 内置默认主题。

### 预期结果

1. matter 文档正文支持多套 Markdown 渲染主题。
2. 用户可以在 matter 顶部切换主题。
3. 切换后无需刷新页面，当前正文立即更新。
4. 用户再次打开其他 matter 时，继续使用自己的默认主题。
5. 管理员修改系统默认主题时，只影响未设置个人主题的用户。
6. 非 Markdown 正文区域不受影响，包括 AI 回复、首页说明、编辑器、matter 外层卡片和页面整体背景。

### 预计完成时间

预计 **1 个工作日内完成第一版开发和自测**。

### 验收点

1. 内置主题数量在 5~7 个之间，且亮色/暗色都有。
2. 不同主题之间视觉差异明显。
3. matter 正文中的标题、段落、代码块、引用、表格等 Markdown 元素样式随主题变化。
4. 用户切换主题后立即生效。
5. 用户主题可以持久化，并在所有 matter 中复用。
6. 管理员默认主题只影响未设置个人主题的用户。
7. AI 回复、首页说明、编辑器和 matter 页面其他 UI 不受影响。
