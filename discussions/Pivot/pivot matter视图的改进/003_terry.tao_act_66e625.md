# Matter 视图改进 — Spec

## 一、查找与筛选功能（全局视图 + 和我有关视图通用）

### 1.1 关键字搜索

| 项目 | 定义 |
|------|------|
| 匹配范围 | 仅 matter 标题，不检索 timeline 内容 |
| 匹配方式 | **子串包含匹配（contains）**，不区分大小写 |
| 生效逻辑 | 与筛选条件取交集，输入即搜（debounce 300ms） |
| 清空 | 输入框右侧提供一键清空按钮 |

> ⚠️ 原始需求写的是"精确匹配"，经与张菠讨论判定为不合理——用户很少能记住完整标题，改为子串包含匹配。

### 1.2 条件筛选

| 维度 | 说明 |
|------|------|
| Owner | 当前 matter 的负责人（owner 字段），支持多选 |
| Status | 支持多选，根据现有Matter状态进行枚举

**交互细节：**
- 筛选器 UI 建议采用 **下拉多选 + 已选 Tag** 形式（类似 Linear 的 filter bar）
- 搜索词 + Owner 筛选 + Status 筛选 三者取交集
- 筛选条件仅在当前会话内生效，刷新后重置（V1 不做持久化）
- **空状态**：交集为空时显示"没有匹配的 matter"占位提示

## 二、"和我有关的视图"改进

### 2.1 "与我有关"的定义

一个 matter 满足以下**任一**条件即出现在此视图中：
1. 我是该 matter 的 owner
2. 我在该 matter 的任意 timeline 文件或评论中被 @mention
3. 我在该 matter 下创建过 timeline 文件或评论


### 2.2 持久化

- 用户阅读后，matter **不从列表中移除**
- 列表长期增长后依赖虚拟滚动保证性能（V1 先做前 200 条分页，后续按需加载）

### 2.3 排序规则

列表分为两个区段，顺序固定：
[未读区 — 按最后更新时间倒序]
[已读区 — 按最后更新时间倒序]

| 项目 | 定义 |
|------|------|
| "时间"含义 | matter 下最新一条 timeline 文件 / 评论的创建时间（`created_at`） |
| "未读"判定 | matter 下存在任何一条我尚未查看的 timeline 文件或评论，即为未读 |
| 未读清除时机 | 用户点击该 matter 所有未读timeline并展开全文时，标记该 matter 所有内容为已读 |
| 状态切换动画 | 阅读后 matter **不立即移动**到已读区，等用户下次进入视图 / 手动刷新时重新排序，避免列表抖动 |
