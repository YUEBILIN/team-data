1. **关于流程引导与 UI 体验**：如果用户不知道如何推进下一步，本质上不是缺乏流程说明文档，而是我们的界面还不够友好，未能自然而然地引导用户。Pivot 如果要成为成功的企业级管理工具，实施门槛必须足够低，绝不能要求每个人先学习软件怎么用，而应该通过界面设计让用户自然地被引导到下一步。Pivot 本身的工作流状态是很自然的（沿着时间线追加工作推动 matter 直至解决），但目前 UI 表现确实比较难用，很多按键功能对中文用户来说不够直观，凭直觉看不出点击后会发生什么。这需要产品侧持续打磨，方向就是大幅提升 UI 的友好度，让用户清晰地理解下一步该干什么。

2. **关于左侧菜单**：左侧菜单太散、帖子太多的问题，我周末已经提了需求，后续会增加“与我相关的信息”（如圈过我、指定过我）的过滤功能，目前应该已经安排有人在开发了。

3. **关于图片与通知功能**：
   - **图片功能**：我希望大家参与讨论。我个人倾向于增加，但必须考虑企业实际接入的 API 能力。大部分企业接入的往往不是 Claude 或 Codex 这种顶级的多模态 AI，如果接的是 DeepSeek 这类非多模态模型，根本无法识别图片。因此，我建议先安排一个测试，看看主流性价比高的国产大模型在识别带图片或其他附件主题时的效果如何，是否足以满足日常需求？如果测试结果理想，我们再考虑增加图片功能。
   - **企业通知**：是否用 Pivot 替代 IM 发布通知，我目前还没有完全想好。建议暂时继续使用 IM 工具发布通知，我们在实际使用中根据情况再做决定。毕竟这目前不是主要矛盾，我个人暂时没想清楚，其他同学如果有好的建议也可以提一提。
