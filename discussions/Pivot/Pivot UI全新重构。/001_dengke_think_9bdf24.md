# Pivot UI全新重构。

## 背景与动机

当前 Pivot 的信息组织方式本质上类似于一个邮件列表——信息以列表形式堆叠，缺乏直观的空间感和上下文关联。对于大部分用户而言，这种交互方式不够友好，也不够直观，尤其在信息量增长后，理解和追溯上下文变得困难。

我们需要一次彻底的界面重构来改善这一体验。

## 核心设计理念：时间线 + 卡片

重构的核心是引入「时间线」概念：

- **所有信息都是时间线上的一张张卡片**，按时间顺序从前往后排列。
- 用户在时间线上**阅读卡片、回应卡片**，以此作为日常工作的基本交互方式。
- 时间线提供了清晰的时间上下文，让信息的产生、流转和回应关系一目了然。

这种设计将 Pivot 从「被动浏览列表」转变为「在时间流中参与对话」，更贴合团队协作的自然节奏。

## 改版范围

本次重构覆盖以下内容：

1. **PC 端界面**：全新的时间线布局与卡片交互设计。
2. **移动端界面**：适配小屏幕的时间线体验，确保移动场景下的可用性。
3. **信息架构重构**：在保留现有全部功能的前提下，对信息展示层进行完整重构——不增减功能，而是改变功能的组织与呈现方式。

## 工作评估

这是一个复杂且完整的工作，涉及信息架构设计、交互设计、前后端适配以及双端开发。预计整体完成时间为 **1.5 周**。

本次重构不仅是视觉层面的改版，更是对 Pivot 核心交互范式的升级，值得在设计和实现上投入足够精力，确保一次到位。
