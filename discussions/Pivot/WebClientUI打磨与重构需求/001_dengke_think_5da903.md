# Pivot Web Client UI 打磨与重构需求

## 背景

当前 Pivot Web Client 的前端界面已经基本可用，但整体还停留在“能用”的阶段，距离“好用、顺滑、稳定、美观”仍有较大差距。现阶段存在的问题主要包括：

- 部分页面或控件在不同设备下适配不稳定，存在显示异常、交互阻塞、无法正常操作的问题
- 移动端和桌面端的交互体验不一致，部分功能在手机上使用成本较高
- 页面层级、信息结构、视觉节奏和反馈机制较弱，影响整体使用体验
- 动效和状态反馈不足，界面切换、生效反馈、操作过程不够自然顺滑
- 当前 UI 风格和组件细节还比较粗糙，需要进行系统性打磨

因此，需要对现有 Pivot Web Client 前端 UI 进行一轮系统性的整理、修复和重构。

---

## 项目目标

本次改造的核心目标是：让 Pivot Web Client 从“基本可用”提升为“稳定、顺滑、清晰、美观、适合持续使用”的产品界面。

重点包括以下三个方向：

### 1. 保证各 UI 元素可正常使用、正常显示

目标是先解决所有基础可用性问题，确保用户在常见设备和浏览器环境下可以稳定完成主要操作，不出现明显阻塞、错位、遮挡、点击无效、功能不可达等问题。

覆盖范围包括：

- PC 端：以 Chrome 为主要支持目标
- 移动端：
  - iOS
  - Android
  - 鸿蒙
- 飞书内置 WebView 场景下的基本可用性

重点检查和修复的问题包括：

- 页面布局错位、溢出、遮挡、宽高异常
- 弹窗、抽屉、底部面板、输入框在移动端的显示和交互问题
- 键盘弹起后输入区域被遮挡、页面高度异常、滚动失效
- 按钮、切换器、列表项、卡片操作区点击不稳定或误触
- 页面切换、返回、展开、收起等核心交互链路中的阻塞问题
- 长文本、超长标题、列表内容较多时的展示与操作问题
- 状态栏、安全区、底部手势区、不同屏幕宽度下的适配问题

### 2. 打磨界面交互体验，增加合理动效和顺滑反馈

目标是在不增加使用负担的前提下，提升 UI 的流畅感、反馈感和操作连续性，让整个产品使用起来更自然。

重点方向包括：

- 为关键交互补充必要的动效和过渡
- 提升点击、展开、切换、弹出、折叠、拖拽等行为的连贯性
- 增强状态变化时的视觉反馈，让用户更容易理解当前系统状态
- 降低界面突变、闪烁、跳动、卡顿和生硬感

建议重点打磨的交互包括：

- 页面进入和退出
- 卡片展开与收起
- 面板弹出、缩小、最大化、关闭
- 列表刷新、内容更新、状态变化
- 按钮点击反馈、悬停反馈、禁用态、加载态
- 输入区聚焦、提交、保存、切换模式时的反馈
- 移动端底部面板、手势拖拽、键盘联动等场景的过渡体验

总体要求是：

- 动效应轻量、清晰、服务于交互理解
- 不做炫技型动画，不影响性能
- 不增加额外认知负担
- 保持移动端和桌面端体验风格统一

### 3. 对界面信息元素进行重新设计和重构，提升美观与使用便捷性

目标是重新梳理页面中的信息层次、操作入口、卡片结构和视觉节奏，使用户更容易理解页面、找到操作入口、快速浏览重点内容。

重点包括：

- 重构页面的信息层级和视觉主次
- 优化列表、卡片、标题区、工具栏、操作区的布局结构
- 提升按钮、标签、状态、提示信息的识别性
- 改善留白、间距、字号、颜色、对齐、边界等细节
- 减少页面噪音，让用户更聚焦于讨论内容和关键操作
- 让常用功能更容易找到、更容易理解、更容易操作

建议重点优化的界面对象包括：

- 线程列表页
- 主题详情页
- 帖子卡片
- 回复卡片
- AI 助手 / AI 总结面板
- 顶部导航与工具区
- 移动端底部交互区域
- 各类弹窗、抽屉、面板、表单输入区
- 状态标签、未读提示、按钮组、提示文案等辅助信息元素

---

## 工作要求

本次改造不应只做“局部修补”，而应以“前端体验整顿”的方式进行，要求团队从以下角度统一推进：

- 先解决可用性问题，再做视觉和体验增强
- 优先处理高频页面和高频操作链路
- 移动端体验和桌面端体验都要覆盖，不可只优化单端
- 设计调整要与实际使用场景结合，不能只追求视觉好看
- 所有改动都应以提升“信息清晰度、操作效率、使用舒适度”为目标
- 不引入明显增加复杂度但收益有限的视觉设计
- 保持整体 UI 风格一致，避免局部重做后风格割裂

---

## 交付预期

本轮改造完成后，应达到以下结果：

- 主流程页面在 PC 和移动端都可以稳定使用，没有明显适配问题
- 用户进行浏览、切换、回复、查看 AI、展开全文等操作时更加顺滑自然
- 页面结构更清晰，重点信息更突出，操作入口更直观
- 整体视觉观感明显提升，从“能用”提升到“好用”
- 前端界面具备继续演进的基础，不再处于零散修补状态

---

## 验收标准

### 可用性验收

- 主要页面在 Chrome 下可稳定使用
- 主要页面在 iOS、Android、鸿蒙移动端浏览器及飞书内置 WebView 下可正常显示和操作
- 不存在明显的按钮不可点、输入框被挡、面板错位、布局溢出、内容遮挡、滚动异常等问题
- 核心链路无阻塞：
  - 浏览主题列表
  - 打开主题详情
  - 浏览帖子与回复
  - 展开/收起全文
  - 打开和使用 AI 面板
  - 输入与提交内容

### 交互体验验收

- 关键操作具备清晰且轻量的动效反馈
- 页面切换、面板展开收起、状态切换无明显突兀感
- 刷新、加载、提交、保存等状态反馈明确
- 移动端操作过程不生硬、不跳动、不闪烁

### 视觉与结构验收

- 页面视觉层级清晰，重点信息突出
- 信息布局更合理，常用操作更易找到
- 卡片、列表、按钮、标签、提示等元素风格统一
- 整体界面观感明显优于当前版本，达到可持续使用的产品水平
