MCP  Matter — 延展优化点验收测试                                                                                                                                        
   
  背景                                                                                                                                                                        
                  
  本次评审同步收口 4 个延展优化点：

  1. MCP 增加 @人 能力
  2. MCP 增加跨帖子引用能力
  3. 调用结果直接可达，用户不再手工复制 URL
  4. 接入端的 MCP 能力面板排版与示例补齐

  下文给出每一项的验收用例和实测结论。

  ▎ 测试范围：内部用户 ~10 人；测试时段：2026-04-29；执行人：yzy。

  ---
  用例 1 · MCP 支持 @人

  ┌──────┬──────────────────────────────────────────────────────────────────────────┐
  │ 项目 │                                   内容                                   │
  ├──────┼──────────────────────────────────────────────────────────────────────────┤
  │ 入参 │ 创建 / 回复 / 评论 时附带 mentions = { targets: [pinyin], say: 一行 DM } │
  ├──────┼──────────────────────────────────────────────────────────────────────────┤
  │ 行为 │ 后端解析 pinyin → open_id；解析失败返回字段级错误，AI 不会静默丢人       │
  ├──────┼──────────────────────────────────────────────────────────────────────────┤
  │ 通知 │ 被 @ 的用户收到飞书 DM + Matter 详情页可见标记                           │
  ├──────┼──────────────────────────────────────────────────────────────────────────┤
  │ 范围 │ create_matter / create_file / add_comment 三个写入工具均支持             │
  └──────┴──────────────────────────────────────────────────────────────────────────┘

  ├─────┼─────────────────────────────────────────┼──────────────────────────────────────┼─────────┤

  ---
  用例 2 · MCP 支持跨帖子引用（verify 闭环）

  ▎ 产品口径：跨帖子引用以 verify 为准 —— 在另一条 matter 里跑通验证即视为需求闭环。其他类型的"引用"按 matter 内 quote / refer 处理，不算跨帖子能力 gap。

  ┌──────┬────────────────────────────────────────────────────────────────────────────┐
  │ 项目 │                                    内容                                    │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 入参 │ create_file 的 refer[]（最多 4 条），verify 类型时同时作为跨 matter 白名单 │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 行为 │ verifications[].target 若位于其他 matter，必须出现在 refer[] 中才被接受    │
  └──────┴────────────────────────────────────────────────────────────────────────────┘

  测试场景

  ┌─────┬─────────────────────────────────────────────────────────────────────────────┬──────────────────────────┬─────────┐
  │  #  │                                    场景                                     │           预期           │  实测   │
  ├─────┼─────────────────────────────────────────────────────────────────────────────┼──────────────────────────┼─────────┤
  │ 2.1 │ 在 A matter 写 verify，target 指向 B matter 的某文件，且该文件在 refer[] 内 │ 接受，记录跨 matter 验证 │ ✅ 通过 │
  ├─────┼─────────────────────────────────────────────────────────────────────────────┼──────────────────────────┼─────────┤
  │ 2.2 │ 同上，但目标文件不在 refer[]                                                │ 拒绝，返回字段级错误     │ ✅ 通过 │
  ├─────┼─────────────────────────────────────────────────────────────────────────────┼──────────────────────────┼─────────┤
  │ 2.3 │ matter 内的 quote / refer 仍按原语义生效                                    │ 引用关系正确             │ ✅ 通过 │
  └─────┴─────────────────────────────────────────────────────────────────────────────┴──────────────────────────┴─────────┘

  结论：跨 matter verify 能走通即视为需求闭环；非 verify 不在本次范围，不算 gap。
