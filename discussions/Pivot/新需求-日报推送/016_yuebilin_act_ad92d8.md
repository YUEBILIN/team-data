## act：完善日报HTML页面版入口与飞书精简推送展示

本次把日报从“飞书卡片里塞完整内容”调整为“飞书推送摘要 + 页面版查看完整日报”的模式，并补齐页面版阅读体验。

### 具体改动

1. **新增日报页面版入口**

   为日报运行记录增加 `/daily-report/runs/:runId` 页面入口，前端通过 run id 读取日报 `debug_json` 中保存的完整 card 内容，支持在页面里查看完整日报。

2. **飞书推送改为精简摘要**

   当日报有页面链接时，飞书卡片只推送 compact 版本：公司日报展示“今日概况 / 需要关注”，个人日报展示“团队动态摘要 / 重点人员动态”，并附带“查看完整日报”按钮跳转页面版。

3. **后端串通 report_url**

   日报任务调度、手动触发、run-now 都会根据 `web_base_url` 生成 `/daily-report/runs/{run_id}` 链接，并传给 runner / card renderer，用于生成飞书按钮和 compact 卡片。

4. **页面版日报增加两种视图**

   页面版支持“完整日报”和“路线图”两个 tab。完整日报保留正文结构；路线图按方向和推进状态拆成表格视图，展示 `Done / Verify / Think` 等分组标签，并对事项列表做了更清晰的编号展示。

5. **修复飞书摘要中的小节编号误导**

   飞书 compact 卡片的“重要关注”抽取关注项时，会去掉行首 `(1)` 这类小节结构编号，避免用户误以为它是事项序号。

### 验证

补充并通过日报渲染相关回归测试，覆盖完整日报链接、compact 飞书卡片、以及“重要关注”不再带出 `(1)` 小节编号的场景。
