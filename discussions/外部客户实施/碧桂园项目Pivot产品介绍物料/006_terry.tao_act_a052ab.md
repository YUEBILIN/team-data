## 三、Pivot 是什么

### 一句话

**Pivot 是企业的"事项级中枢神经"。**

人体的神经系统不替代心脏、肝脏、四肢，但它把这些器官调度成一个有意识、能反应、能学习的整体。Pivot 在企业里做的就是这件事。

### 它做三件事

1. **接收**——每件事的全过程信息（来自系统、对话、人工补录）都汇入 Pivot
2. **判断**——AI 持续追踪每件事现在卡在哪、谁该做什么、什么时候做
3. **沉淀**——所有过程留痕，可追溯、可复盘、可作为治理证据

### 一个心智锚点

> 在过去：一件事出问题，要去翻聊天记录、查工单、问当事人。
>
> 在 Pivot 里：每件事都有自己的"档案 + 跟进过程 + 当前判断"，一眼就能看清。

## 四、AI Agent 是什么

### 一句话

**AI Agent 是 Pivot 的"执行与反馈网络"——把大脑的判断送到具体系统、具体人，再把执行结果带回大脑。**

### 它的角色

- **不是孤立的 AI 工具**——不是某个环节里独立运行的小工具
- **不是替代器**——不取代碧桂园已建/在建的业务系统和 AI 能力
- **是连接器**——让 Pivot 的判断真正跑通到现有系统、现有岗位、现有渠道里

### 一个例子

> Pivot 判断："这位租客的报修工单需要在 24 小时内派工"——
>
> AI Agent 立刻：
>
> - 在服务商系统下派单
> - 通过微信通知租客上门时间
> - 通知房东
> - 12 小时内若无响应，自动升级到客服主管

## 五、我们提供什么服务

### 第一层：Pivot 核心系统（标准产品）

- **形态**：成熟、标准化的产品，一次引入，长期使用
- **角色**：碧桂园的"智慧大脑"
- **客户拿到**：一个开箱可用的事项调度中枢
- **能力固定，不需要定制**

### 第二层：AI Agent 定制化开发（项目服务）

- **形态**：根据碧桂园的业务流程和系统现状，逐个定制
- **角色**：Pivot 大脑的"手脚 + 感官"
- **客户拿到**：与碧桂园业务流深度契合的执行网络
- **按业务优先级分阶段交付，可滚动扩展**

## 六、如何与碧桂园系统融合

### 总原则：不取代，融合

- 碧桂园的**交易、履约、服务**三大业务系统按规划继续建设
- 已规划的**局部 AI 能力**（智能匹配、估价、3D、知识库、垂直大模型）继续推进
- Pivot **定位中枢指挥**——它在这些之上，让它们被统一调度

### 融合的三种方式

| 方式     | 含义                                           | 用一句话讲                   |
| -------- | ---------------------------------------------- | ---------------------------- |
| **复用** | 碧桂园已有系统能完成的信息流，我们直接接入     | 你们已经有的，我们不重做     |
| **适配** | 碧桂园系统具备能力但格式不统一，加一层轻量翻译 | 你们差一点点的，我们补一层   |
| **补齐** | 碧桂园暂时没有的能力，由 AI Agent 定制开发     | 你们暂时没有的，我们造给你们 |

### 一个重要保证

> **碧桂园系统在建是常态，不影响起步。** 我们可以从最关键的 1-2 条业务流先跑起来，逐步扩展——你们的系统建到哪，我们的 Agent 就跟到哪。

## 七、对碧桂园的业务价值

### 核心价值：**从"局部 AI 工具"升级到"体系级事项调度"**

#### 现状

碧桂园已规划的 AI 能力（智能匹配、估价、3D、知识库、大模型）都是优秀的**局部工具**。它们解决的是：

- 这个环节用 AI 做得更快
- 那个环节用 AI 做得更准

但**跨环节、跨系统、跨角色**的事情，依然散落——

- 没有统一的"这件事现在到哪了"
- 没有统一的"谁该做什么、何时之前"
- 没有统一的"出问题时回看一切的过程"

#### Pivot 带来的

每一件横跨多个系统、多个角色的事，都被统一调度：

| 维度             | 局部 AI 工具 | Pivot 调度体系         |
| ---------------- | ------------ | ---------------------- |
| **覆盖范围**     | 单环节内     | 跨系统、跨角色、跨阶段 |
| **是否有人负责** | 不一定       | 必有责任人             |
| **过程可见性**   | 散落         | 统一时间线             |
| **结果是否收口** | 不一定       | 必须正式收口           |
| **可追溯证据**   | 部分         | 全程留痕               |

#### 一句话差异

> **从"环节里有 AI"，升级到"业务运转方式被 AI 重新组织"。**
>
> 这一点正好对应碧桂园文档里反复强调的——全程透明可追溯、责任人明确、超时自动提醒、争议有据可查、信誉评分可信。Pivot 是支撑这些治理目标真正落地的**底层调度能力**。

## 八、场景验证 — A：租客报修

`[视觉建议：横向流程图，从微信消息开始，经 AI Agent → Pivot → AI Agent → 各方触达]`

| 阶段       | 发生了什么                                                   |
| ---------- | ------------------------------------------------------------ |
| 起点       | 租客在微信发消息："卫生间漏水了，下午能修吗"                 |
| 进入 Pivot | AI Agent 抓取关键信息（房源、租约、客户身份），把这件事录入 Pivot |
| 大脑判断   | Pivot 创建事项「卫生间漏水-单元 305」；AI 整理出问题描述与处理建议；指派给维修主管，建议 24 小时内完成 |
| 指令下发   | AI Agent 在服务商系统下派单 + 微信回复租客上门时间 + 通知房东 |
| 持续追踪   | Pivot 监听 12 小时无响应自动升级到客服主管                   |
| 验收回流   | 服务商完工，AI Agent 录入完成确认；租客评价"通过"            |
| 正式收口   | 事项标记为已完成；处理经验沉淀，下次同类问题更高效           |

> **价值看点**：传统模式下，这件事散落在微信、客服系统、服务商系统、房东电话四个地方；Pivot 里它是**一条完整时间线 + 一个负责人 + 一个明确结果**。

## 九、场景验证 — B：买卖意向跟进

| 阶段       | 发生了什么                                                   |
| ---------- | ------------------------------------------------------------ |
| 起点       | 客户在 C 端 AI 对话明确表达买房需求                          |
| 进入 Pivot | AI Agent 把意向事件 + 对话内容送入 Pivot                     |
| 大脑判断   | Pivot 创建事项「客户王某-改善型买房意向」；调用 AI 估价、智能匹配整理推荐方案；指派经纪人小张，3 天内带看 |
| 指令下发   | AI Agent 通知经纪人 + 协调带看时间                           |
| 持续追踪   | Pivot 监听 7 天无进展自动提醒                                |
| 反馈回流   | 经纪人补录带看反馈；Pivot 判断进入合同流程                   |
| 推进闭环   | AI Agent 触发意向合同生成                                    |
| 正式收口   | 事项收口；客户偏好沉淀，反哺后续匹配                         |

> **价值看点**：碧桂园已规划的 AI 估价和智能匹配在这里**被 Pivot 当作工具调用**——它们的能力被放进一条完整的客户跟进流程里，而不是孤立运行。

## 十、场景验证 — C：房源造假风控立案（横跨 3 系统）

> **这是传统设计最容易掉链子的场景，也是 Pivot 价值最突出的地方。**

| 阶段       | 发生了什么                                                   |
| ---------- | ------------------------------------------------------------ |
| 起点       | 信任系统识别可疑房源，触发风控告警                           |
| 进入 Pivot | AI Agent 把风控事件送入 Pivot，自动创建事项                  |
| 联动指令   | AI Agent 同时联动：房源系统暂停发布动作 + 履约系统冻结相关待签合同 |
| 协同       | 法务被自动拉入事项的讨论流                                   |
| 全局监督   | Pivot 调度台的"流程风险"分组让运营负责人一眼看到全部立案     |
| 正式收口   | 事项关闭，处置经验沉淀为风控规则                             |

> 整件事涉及 **4 个系统、3 个角色**，但只有**一个统一的事项视图**把它串起来。
>
> **这是 Pivot 不可替代的位置——单点 AI 工具做不到的横跨调度。**

## 十一、下一步

### 内部讨论明确的三件事

1. **试点选哪条业务线？**
   - 建议优先选事项集中、横跨系统多、价值明显的——例如**租后报修**或**风控立案**
2. **现有系统在哪些信息流上已具备能力？**
   - 这决定了哪些走"复用"、哪些需要"补齐"
3. **哪些角色需要尽早纳入？**
   - 建议先内部业务人员进调度台，再扩 B 端经纪人

### 我们下一步动作

- 基于你们的反馈，提供更细化的合作方案：覆盖范围、推进节奏、人力投入预估
- 基于贵方业务专家的分析，配合做一次小规模试点，验证价值后再扩展
