# 优化战报信息展示：按人聚合与优先级排序

## 背景
目前 Pivot 项目的战报包含了丰富的项目动态，但团队日常打开频率有下降趋势。初步反馈表明，这并非因为大家不需要战报，而是信息呈现方式过于平铺直叙，缺乏针对性的过滤与聚合机制。

## 核心痛点
用户每次打开战报，最迫切想确认的通常只有三件事：
- 今天有没有人 @ 我？
- 我负责的 matter 有没有关键状态变化？
- 有没有等待我 review 的结果？

但目前这些高优先级信息完全混在大量普通动态流中。用户必须通读或手动滑动筛选，自行判断哪些与自己相关、哪些需要立即处理。这种“手动分拣”的体验增加了阅读成本，导致信息触达效率打折。

## 待讨论与探索
为提升战报的实用性与阅读体验，建议从以下方向展开探讨：
1. **个性化聚合视图**：是否可增加“我的关注”或“待办动态”面板，将 @ 提及、Owner 变更、Pending Review 等信息自动聚合置顶？
2. **优先级定义与标记**：如何科学划分信息权重？是否需要在 UI 层引入视觉分级（如高亮/折叠/标签），或在数据层增加优先级字段？
3. **技术可行性与成本**：前端评估交互改造与渲染性能；同时需确认底层数据结构是否支持按用户、按状态、按关系进行实时过滤。

## 期望推进事项
- **前端/UX**：输出初步交互方案，评估实现复杂度。
- **数据/后端**：盘点战报现有字段，明确可提取的“高优先级信号”。
- **全员**：补充更多高频使用场景或痛点，协助收敛 MVP 范围。

欢迎大家结合实际使用体验提出建议，或推荐更优的信息组织方式。
